# ADR-0017: Выбор Frontend фреймворка

## Статус
**Accepted** — принято

## Контекст

Нужно создать Web Dashboard для Obsidian Task Manager. Dashboard будет:
- Отображать задачи и проекты
- Позволять создавать/редактировать/удалять
- Фильтровать и искать
- Показывать статистику

Рассматриваем три варианта: React, Vue, Vanilla JS.

## Критерии выбора

1. **Обучаемость** — насколько легко изучить
2. **Экосистема** — доступность библиотек и компонентов
3. **Совместимость** — работа с Obsidian Plugin в будущем
4. **Поддержка** — документация, сообщество
5. **Карьера** — востребованность на рынке

## Рекомендация

**Рекомендую: Вариант A — React + Vite + TypeScript**

### Обоснование

1. **Максимальная совместимость с Obsidian Plugin**
   - Obsidian Plugin API написан на TypeScript
   - React-подобный подход в UI компонентах
   - Переиспользование знаний

2. **Лучшая экосистема**
   - shadcn/ui — готовые красивые компоненты
   - TanStack Query — кэширование API запросов
   - Zustand — простой state management

3. **Карьерная ценность**
   - React — #1 по количеству вакансий
   - Полезный навык для резюме

4. **Учебная ценность**
   - Изучение React — инвестиция в будущее
   - Применим в других проектах

### Стек для проекта

```
Frontend:
├── Vite (сборщик)
├── React 18 (UI библиотека)
├── TypeScript (типизация)
├── TailwindCSS (стили)
├── shadcn/ui (компоненты)
├── TanStack Query (API запросы)
├── React Router (навигация)
└── Zustand (состояние)
```

## Альтернатива

Если приоритет — **скорость обучения**, выбрать Vue:
- Проще синтаксис
- Документация на русском
- Быстрее начать

## Решение

**Выбран: React + TypeScript**

- [x] React + TypeScript — максимум возможностей
- [ ] Vue + TypeScript — проще для старта
- [ ] Vanilla JS — минимум зависимостей
